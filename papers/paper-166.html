Introduction

이 논문은 무엇을 다루나요?

DRAM 프로세스 기술이 축소되고 DRAM 밀도가 계속 증가함에 따라 DRAM 오류는 현대 데이터 센터의 주요 관심사가 되었습니다. 일반적으로 데이터 센터에서는 단일 오류 수정 이중 오류 감지(SECDED) 코드가 있는 메모리 시스템을 채택했습니다. 그러나 메모리 시스템이 더욱 취약해짐에 따라 SECDED 코드는 DRAM 신뢰성 요구를 충족시키기에 충분하지 않습니다. 데이터 센터의 서버는 강력한 ECC 체계를 사용하지만 이러한 ECC 체계는 상당한 성능 및/또는 스토리지 오버헤드를 초래합니다. 본 논문에서는 기존 SECDED 코드보다 성능 오버헤드가 적고 스토리지 오버헤드 없이 강력한 오류 정정성을 제공하는 비용 효율적인 메모리 보호 방식인 Stealth ECC를 제안합니다. 데이터 너비(좁은 너비 또는 전체 너비)에 따라 Stealth ECC는 ECC 구성표를 적응적으로 선택합니다. 폭이 좁은 값의 경우 Stealth ECC는 MSB 측에 0 대신 더 많은 패리티 비트를 저장하여 다중 비트 오류 정정 기능을 제공합니다. 또한, x4 DRAM 칩 사이에 비트 단위로 인터리빙된 데이터 배치를 통해 Stealth ECC는 좁은 폭 값에 대한 단일 DRAM 칩 오류에 강력합니다. 반면, 전체 폭 값의 경우 Stealth ECC는 SECDED 코드를 채택하여 기존 SECDED 코드에 필적하는 DRAM 신뢰성을 유지합니다. 결과적으로, 좁은 폭 값의 신뢰성 향상 덕분에 Stealth ECC는 전반적인 DRAM 신뢰성을 향상시키는 동시에 무시할 수 있는 성능 오버헤드와 스토리지 오버헤드를 발생시키지 않습니다. 시뮬레이션 결과, Stealth ECC는 기존 SECDED 코드에 비해 시스템 오류(DRAM 오류로 인한) 확률을 평균 47.9% 줄였으며 성능 오버헤드는 0.9%에 불과한 것으로 나타났습니다.
