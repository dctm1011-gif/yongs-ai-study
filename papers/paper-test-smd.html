Figure 갤러리

이 그림은 CPU에서 메모리 컨트롤러를 거쳐 DRAM 칩까지 계층적으로 연결된 전체 메모리 시스템 구조를 보여줘요. DRAM 랭크(Rank)는 다중 서브어레이(Subarray)로 구성되며, 각 서브어레이는 로우 버퍼(Row Buffer)를 통해 행 데이터를 캐싱하는 방식이에요. 가장 핵심인 DRAM 셀 부분(우측 원형)은 액세스 트랜지스터와 커패시터로 이루어진 1T1C 구조로, 워드라인(Wordline)과 비트라인(Bitline)을 통해 데이터를 읽고 쓰는 구조예요. 논문의 Self-Managing DRAM 기법은 이러한 기존 DRAM
이 그림은 DRAM 내부에서 자율적으로 로우 해머 공격을 방어하기 위한 Lock Region 제어 구조를 보여줘요. ACT(활성화) 및 유지보수 연산(Maintenance Operation)을 Lock Controller가 관리하며, Global Row Latcher를 통해 특정 로우에 접근하는 메인터넌스 작업을 Lock Region 단위로 제어해요. 핵심은 Row Address를 Global Row Latcher로 라우팅하고, Lock Region Bitvector(LRB)의 N비트로 각 Lock Region을 개별 제어함으로써, 하드웨어 오버헤드를 최소화하면서도 in-DRAM에서 리프레시와 로우 해머 방어를 동시에 수행하는 구조예요.
이 그림은 DRAM의 자율 관리 시스템에서 ACT(Activate) 명령어 실패 후의 재시도 메커니즘을 보여주는 타이밍 다이어그램이에요. 메모리 컨트롤러(MC)가 ACT_NACK을 받으면 ARI(ACT Retry Interval) 동안 대기했다가 다시 ACT를 시도하는데, 이 기간 동안 다른 Lock Region에 대해 RCD(Row to Column Delay) 시간의 RD와 PRE 작업을 병렬로 수행할 수 있어요. 핵심은 자동 리프레시나 RowHammer 방어 등으로 인한 ACT 지연 시간을 활용하여 메모리 대역폭 낭비를 최소화하는 자율 제어 회로 구조에요.
이 그림은 DRAM의 자율 관리형 리프레시 메커니즘을 보여주는 제어 흐름도예요. PRC(Pending Refresh Counter)가 tREFI마다 증가하다가 임계값을 초과하면, LRC(Lock Region Counter)에 해당하는 메모리 영역을 잠금하고 해당 행들을 우선적으로 리프레시하는 구조예요. 핵심은 RowHammer 공격으로 인한 데이터 오염을 방지하면서도 리프레시 오버헤드를 최소화하기 위해, 접근 빈도가 높은 행 주소(RAC)가 집중된 영역을 동적으로 감지하고 관리한다는 점이에요. 즉, in-DRAM에서 하드웨어 카운터를 통해 위협 영역을 자동으로 식별·보호
이 그림은 DRAM의 자율적 갱신(Refresh) 및 행-해머(Row Hammer) 방어를 위한 제어 로직을 보여주는 상태 머신이에요. PRC(Pending Refresh Counter)가 증가하여 임계값을 초과하면, RCC(Refresh Cycle Counter)와 LRC(Lock Region Counter), RAC(Row Address Counter)를 활용해 특정 메모리 영역을 선택적으로 잠금하고 갱신하는 프로세스를 진행합니다. 핵심은 Bloom Filter(BF)를 통해 빈번하게 접근되는 행들을 추적하고, 이들을 우선적으로 표시(Mark)하여 메모리 안정성을 보장하면서도 갱신 오버헤드를 최소화하는 데에 있어요. 이를 통해 RowHammer 공격으로부터의 보안성과 자
