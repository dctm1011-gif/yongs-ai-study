Phase Interpolator-Based Clock and Data Recovery With Jitter Optimization

PLL-DLLProcess
원문 보기 → PDF →

위상 보간기(PI) 기반 CDR 회로의 지터를 체계적으로 분석하고 최적화하는 새로운 방법론을 제시한 논문이에요.

Introduction

왜 PI 기반 CDR의 지터 최적화가 고속 직렬 인터페이스에서 중요한가요?

고속 직렬 인터페이스(HSSI)가 발전함에 따라 데이터 복구 회로(CDR)의 지터 성능은 시스템 신뢰성을 결정짓는 핵심 요소가 되었어요. 특히 메소크로너스(mesochronous) 환경이나 다발성 인터페이스에서는 수신 측에서 클럭과 데이터를 정확하게 정렬하는 것이 매우 까다로운 과제예요. 전통적인 PLL 기반 CDR은 루프 대역폭 설계에 제약이 많고, 빠른 위상 추적에 한계가 있어 고속 환경에서 성능이 저하되는 문제가 있어요. 이를 극복하기 위해 위상 보간기(PI) 기반 CDR이 주목받고 있지만, PI 자체의 비선형성과 양자화 노이즈가 전체 시스템 지터에 미치는 영향을 정확히 모델링하고 최적화하는 방법론은 아직 충분히 연구되지 않았어요.

PI 기반 CDR은 외부에서 공급받은 다위상 클럭을 보간하여 위상을 정밀하게 조절하는 구조로, PLL 기반 대비 빠른 위상 추적과 설계 유연성을 제공해요. 그러나 PI의 위상 분해능, 루프 필터 파라미터, 디지털 제어 루프의 비트 폭 등 다양한 설계 변수들이 복잡하게 얽혀 지터 성능에 영향을 미쳐요. 특히 이중 루프(dual-loop) 구조에서는 주파수 추적 루프와 위상 정렬 루프 간의 상호작용이 지터 특성을 더욱 복잡하게 만들어요. 따라서 체계적인 지터 분석 방법론 없이는 설계 최적화가 매우 어렵고 반복적인 시뮬레이션에 의존할 수밖에 없는 상황이에요.

논문의 출발점

PI 기반 CDR에서 지터 성능을 결정하는 다양한 설계 변수들을 체계적으로 분석할 수 있는 방법론이 부재하다는 점이 핵심 문제예요. 이를 해결하기 위해 본 논문은 지터 기여 요소를 수식적으로 모델링하고, 이를 기반으로 설계 최적화를 수행하는 통합적 접근법을 제안해요.

Proposed Method

어떻게 지터 분석 방법론을 설계에 적용하여 CDR 성능을 최적화했나요?

본 논문은 PI 기반 CDR의 지터를 분석하기 위해 각 구성 요소별 지터 기여분을 분리하여 모델링하는 방법론을 제안해요. 주요 지터 소스로는 PI의 양자화 노이즈, 루프 필터의 디지털 누산기 노이즈, 입력 기준 클럭 지터, 그리고 회로 열잡음 등이 포함돼요. 각 요소의 전달 함수를 CDR 루프 모델에 적용하여 출력 지터에 대한 기여도를 정량화하고, 이를 최소화하는 설계 파라미터 조합을 도출해요. 이 방법론은 단순한 시뮬레이션 반복이 아닌, 해석적 모델을 기반으로 최적 설계점을 사전에 예측할 수 있게 해줘요.

회로 구현 측면에서는 TSMC 65nm CMOS 프로세스 기반의 8비트 이중 루프 CDR을 설계하였으며, 새로운 PI 토폴로지를 적용해 위상 분해능을 확장했어요. 제안된 PI는 기존 구조 대비 선형성이 향상되어 위상 보간 오차를 줄이고 양자화 노이즈를 억제하는 효과를 가져요. 이중 루프 구조는 내부 루프에서 위상 정렬을, 외부 루프에서 주파수 오프셋 추적을 담당하며 5.83Gbps에서 500ppm의 주파수 오프셋을 추적할 수 있어요. 검증은 Verilog-AMS 기반 지터 생성기와 트랜지스터 레벨 CDR 회로를 Cadence AMS 시뮬레이터에서 공동 시뮬레이션하는 방식으로 수행되었어요.

핵심 아이디어

지터 기여 요소를 해석적으로 모델링하여 설계 파라미터를 사전에 최적화하는 방법론과, 선형성이 향상된 새로운 PI 토폴로지의 결합이 이 논문의 핵심 혁신이에요. Verilog-AMS와 트랜지스터 레벨 혼합 시뮬레이션을 통해 실제 지터 환경을 정밀하게 재현하여 검증 신뢰성도 크게 높였어요.

Results

어떤 성능 결과로 제안된 방법론과 설계의 우수성을 입증했나요?

제안된 CDR은 5.83Gbps 동작 속도에서 10^-10 BER 이하의 비트 오류율을 달성하며 안정적인 데이터 복구 성능을 입증했어요. 1V 공급 전압에서 14.2mW의 전력 소비로 전력 효율도 양호한 수준이에요. 주파수 추적 범위는 500ppm으로, 메소크로너스 및 다발성 HSSI 수신기 환경에서 요구되는 주파수 오프셋 보상 요건을 충족해요. Verilog-AMS 혼합 시뮬레이션을 통한 검증 결과, 제안된 지터 분석 방법론이 예측한 최적 설계점이 실제 시뮬레이션 결과와 잘 일치함을 확인했어요.

HBM 설계에 어떻게 적용할 수 있나요?

HBM 및 고속 메모리 인터페이스 설계자 관점에서 이 논문은 PI 기반 CDR의 지터 최적화 방법론을 체계적으로 제시하고 있어 매우 유용해요. HBM 인터페이스에서는 다수의 데이터 레인이 병렬로 동작하며 각 레인의 위상 정렬이 중요한데, PI 기반 CDR은 다위상 클럭을 활용하여 레인별 독립적인 위상 조정이 가능하다는 장점이 있어요. 특히 이중 루프 구조는 HBM에서 요구되는 주파수 추적과 위상 정렬을 분리하여 처리함으로써 전체 시스템 안정성을 높이는 데 기여할 수 있어요. 제안된 지터 분석 방법론을 HBM 설계에 적용하면 수십 개 레인의 지터 버짓을 체계적으로 관리하는 데 활용할 수 있어요.

65nm CMOS 프로세스 기반의 설계이지만, 제안된 방법론과 PI 토폴로지의 개념은 더 선진 공정 노드로 스케일링하는 데도 적용 가능해요. HBM3/HBM3E와 같은 최신 규격에서는 데이터 레이트가 더욱 높아져 지터 마진이 더욱 빡빡해지므로, 이런 분석 방법론의 중요성은 더 커질 것으로 예상돼요. 또한 500ppm 주파수 오프셋 추적 기능은 호스트와 DRAM 다이 간의 클럭 도메인 차이를 보상하는 데 실용적이에요. Verilog-AMS 기반 혼합 시뮬레이션 방법론은 HBM 설계 검증 플로우에 직접 통합하여 활용할 수 있어요.

주의할 점

본 논문의 검증이 시뮬레이션 레벨에 국한되어 있어 실리콘 측정을 통한 실제 성능 확인이 이루어지지 않은 점은 주의가 필요해요. 65nm 공정 기반 설계이므로 HBM에 실제 적용되는 선진 공정(예: 10nm 이하)에서의 성능 변화와 추가적인 기생 효과에 대한 재검증이 필요해요. 또한 5.83Gbps의 동작 속도는 최신 HBM3E 규격(~9.6Gbps 이상)에 비해 낮으므로, 더 높은 데이터 레이트로의 확장 가능성에 대한 추가 연구가 요구돼요.
