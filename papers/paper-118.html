DBB-ECC: Random Double Bit and Burst Error Correction Code for HBM3

원문 보기 →

HBM3의 버스트 오류와 랜덤 이중 비트 오류를 추가 패리티 비트 없이 동시에 잡아내는 DBB-ECC가 등장했어요!

Introduction

왜 HBM3에서 기존 ECC만으로는 더 이상 충분하지 않을까요?

DRAM 기술이 미세공정으로 계속 축소되면서 셀 간 간섭, 전하 누설, 방사선 영향 등으로 인한 비트 오류가 급격히 증가하고 있어요. 이를 해결하기 위해 최신 DRAM 공급업체들은 칩 내부에 온다이 ECC(On-Die ECC)를 탑재하는 방향으로 전환했어요. HBM3은 특히 고대역폭과 고집적을 동시에 요구하는 메모리로, 신뢰성 확보가 시스템 전체 안정성에 직결되는 핵심 과제예요.

HBM3에서는 버스트 오류 정정을 위해 단일 기호 정정(SSC) 리드 솔로몬(RS) 코드가 활용되어 왔어요. SSC RS 코드는 연속된 비트들이 한꺼번에 틀리는 버스트 오류에 강점을 가지지만, 공격적인 공정 미세화로 인해 무작위로 분산된 랜덤 이중 비트 오류(Random Double Bit Error)가 빈번하게 발생하는 현실에서는 한계를 드러내요. 두 가지 오류 유형을 모두 커버하려면 기존 코드 구조를 넘어서는 새로운 접근이 반드시 필요해요.

논문의 출발점

HBM3 환경에서 버스트 오류와 랜덤 이중 비트 오류가 동시에 발생할 수 있는데, 기존 SSC RS 코드는 이 두 가지를 함께 처리하지 못해요. 패리티 비트나 스토리지 오버헤드를 늘리지 않으면서도 두 유형의 오류를 모두 정정하는 방법이 이 논문의 핵심 문제 정의예요.

Proposed Method

DBB-ECC는 어떻게 두 가지 오류를 추가 비용 없이 동시에 정정하나요?

DBB-ECC는 기존 SSC RS 코드의 신드롬 구조를 재활용하면서 이중 비트 오류의 신드롬 패턴이 단일 기호 오류의 패턴과 구별 가능하다는 수학적 특성을 핵심 아이디어로 삼아요. 즉, 기존 SSC 코드에서 생성되는 신드롬 값을 추가적인 패리티 비트 없이도 랜덤 이중 비트 오류 검출 및 정정에 활용할 수 있어요. 오류 유형을 판별하는 분류 로직을 디코더 앞단에 배치하여, 버스트 오류인지 이중 비트 오류인지를 먼저 식별한 후 각 경로로 처리하는 구조를 취해요.

이중 비트 오류의 신드롬 패턴 특성을 활용하면 디코더 내부 연산 복잡도도 줄일 수 있어요. 일반적인 이중 오류 정정 코드는 복잡한 다항식 풀이 과정을 요구하지만, DBB-ECC는 신드롬 패턴 매칭 방식으로 이를 단순화했어요. 이를 통해 디코더 회로 면적, 지연 시간, 전력 소비 모두를 기존 솔루션 대비 낮출 수 있으며, HBM3의 타이트한 타이밍 요구사항을 만족시킬 수 있어요.

핵심 아이디어

SSC RS 코드의 신드롬 정보만으로도 랜덤 이중 비트 오류의 위치를 특정할 수 있는 신드롬 패턴 구조적 특성을 발견하고, 이를 디코더 설계에 직접 적용한 것이 DBB-ECC의 핵심 혁신이에요. 추가 패리티 비트 없이 기능을 확장했기 때문에 스토리지 오버헤드는 기존 SSC와 동일하게 유지돼요.

Results

DBB-ECC는 실제로 기존 방식보다 얼마나 더 효율적인가요?

실험 결과에 따르면 DBB-ECC는 기존 SSC RS 코드와 동일한 스토리지 오버헤드를 유지하면서 랜덤 이중 비트 오류 정정 기능을 추가로 확보했어요. 동일 수준의 정정 능력을 제공하는 기존 솔루션들과 비교할 때 회로 구현 오버헤드(면적, 지연, 전력) 모두에서 우위를 보였어요. 이는 HBM3처럼 면적과 전력에 극도로 민감한 고집적 메모리 시스템에 매우 적합한 결과예요.

HBM 설계에 어떻게 적용할 수 있나요?

HBM 설계자 입장에서 DBB-ECC의 가장 큰 매력은 기존 SSC RS 코드 인프라를 크게 변경하지 않고도 정정 능력을 확장할 수 있다는 점이에요. 온다이 ECC를 설계할 때 스토리지 오버헤드는 유효 용량 손실과 직결되기 때문에, 패리티 비트를 늘리지 않고 기능을 강화하는 방식은 수율과 원가 모두에 긍정적으로 작용해요. 특히 HBM3처럼 다수의 다이를 적층하는 구조에서는 각 다이별 ECC 회로의 면적과 전력이 누적되기 때문에, 구현 오버헤드 절감 효과가 배가돼요.

공정 미세화가 계속될수록 랜덤 비트 오류와 버스트 오류가 혼재하는 환경이 더욱 심화될 것이므로, DBB-ECC와 같이 복합 오류 유형을 처리하는 ECC 아키텍처는 HBM4 이후 세대에서도 유효한 설계 방향성을 제시해요. 수율 관점에서도, 기존에는 이중 비트 오류로 인해 폐기되던 다이가 DBB-ECC 적용 시 정상 동작 가능한 범위로 구제될 수 있어 실질적인 수율 향상 효과를 기대할 수 있어요.

주의할 점

DBB-ECC는 랜덤 이중 비트 오류와 단일 기호 오류를 동시에 정정하지만, 세 개 이상의 랜덤 비트 오류나 두 기호 이상의 버스트 오류가 동시에 발생하는 경우에는 정정 능력의 한계가 존재해요. 또한 신드롬 패턴 기반의 오류 유형 분류 로직이 엣지 케이스에서 오분류할 가능성에 대한 충분한 검증이 실제 양산 적용 전에 필요해요.
